# Decision Records

This directory records rationale for durable OMYM2 product and architecture
decisions.

Decision records are not active specifications. Current rules live in
`ARCHITECTURE.md` and the relevant documents under `docs/`.

Read a decision record only when changing, challenging, or explaining the related
rule.

Create a decision record only when the decision affects future implementation
work and rejected alternatives are likely to be reconsidered.

Do not create decision records for routine implementation details, small
refactors, test-only changes, or temporary workarounds.

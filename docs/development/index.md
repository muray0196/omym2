# Development

This folder contains the development harness, test policy, and benchmark procedure.

* [Development Harness](harness.md) - Specifies dependency setup, current quality gates, Codex completion validation, checks.sh, Windows desktop CI expectations, suppressions, and runtime boundaries.
* [Testing](testing.md) - Defines OMYM2's Python, frontend, browser, architecture, integration, retained-object filesystem, native Windows, provider/companion/unprocessed fixture, rollback, and CI test policy.
* [Pipeline Performance Benchmark](benchmarks.md) - Defines the reproducible end-to-end pipeline benchmark, its dataset, release measurement record, provider-cache qualification, and trust-stat comparison procedure.
* [Windows Desktop Packaging](desktop-packaging.md) - Defines the Windows x64 desktop build, renderer and artist-naming distribution boundaries, native filesystem and package-smoke CI evidence, data lifecycle, licensing, and signing.
